Cliente:
	Instalación:
		# apt-get install openssh-client

		Nota: normalmente `openssh-client` viene preinstalado en casi todas las distribuciones de GNU/Linux.

	Uso:
		$ ssh [-b <IP origen>] [-C] [-p <puerto>] <usuario>@<host>

		Parámetros:
			· `b <IP origen>`: En caso de que se tengan varias direcciones IP, es posible elejir con cual de ellas realizar la conexión.
			· `C`: Compresión, útil solo para redes lentas.

	Montaje remoto:
		Es posible montar una carpeta remota por medio de `sshfs` y tener la capacidad de aplicar operaciones comunes (copiar, pegar, eliminar, etc..) sin utilizar la línea de comandos.

		Instalación:
			# apt-get install sshfs

		Montaje:
			$ sshfs <usuario>@<host>:[<carpeta>] <punto de montaje> [-C] [-p <puerto>]

			Parámetros:
				· `C`: Compresión, útil solo para redes lentas.

		Desmontaje:
			$ fusermount -u <punto de montaje>

	Llaves SSH:
		Permiten establecer conexiones sin tener que escribir la contraseña en cada intento, para crearlas se deben usar los siguientes comandos:

			$ ssh-keygen -t rsa
			Enter file in which to save the key (~/.ssh/id_rsa): [Enter]
			Enter passphrase (empty for no passphrase): [Enter]
			Enter same passphrase again: [Enter]
			$ mkdir -p ~/key/.ssh
			$ cp ~/.ssh/id_rsa.pub ~/key/.ssh/authorized_keys
			$ scp -ro PubkeyAuthentication=no ~/key/.ssh <usuario>@<host>:
			$ rm -rf ~/key/.ssh

	SSH Reverso:
		ssh -R <puerto destino>:localhost:<puerto SSH> <usuario destino>@<host destino>

Servidor:
	Instalación:
		# apt-get install openssh-server

	Configuración:
		1. Editar el archivo de configuración:
			# <editor> /etc/ssh/sshd_config

			Protocol 2
			AddressFamily inet

			PermitRootLogin no

			MaxAuthTries 1

			DebianBanner no
			PrintMotd no
			PrintLastLog no
			Compression no



			HostKey /etc/ssh/ssh_host_rsa_key
			HostKey /etc/ssh/ssh_host_dsa_key
			HostKey /etc/ssh/ssh_host_ecdsa_key
			HostKey /etc/ssh/ssh_host_ed25519_key
			HostbasedAuthentication no
			SyslogFacility AUTH
			LogLevel INFO
			StrictModes yes
			ChallengeResponseAuthentication no
			Subsystem sftp /usr/lib/openssh/sftp-server
			UsePAM yes

		2. Reiniciar SSH:
			# /etc/init.d/ssh restart

		Opciones disponibles:
			Conectividad:
				· "Port": Puerto usado por el servidor. Valor por defecto: `22`.
				· "AddressFamily": Tipo de direcciones a soportar. `inet` para IPv4, `inet6` para IPv6 y `any` para cualquiera. Valor por defecto: `any`.
				· "ListenAddress": Direcciones locales permitidas, por defecto están todas habilitadas.

			Filtros:
				Todas las opciones después de un filtro pertenecerán a él hasta encontrar otro filtro.

				· "Host": Filtro por direcciones, se pueden usar los comodines `*` y `?` de los patrones GLOB además de `!` para negar alguna expresión, para especificar múltiples se deben separar por espacios.

				· "Match user": Filtro por usuarios, se pueden usar los comodines `*` y `?` de los patrones GLOB además de `!` para negar alguna expresión, para especificar múltiples se deben separar por comas.

			Restricción:
				· "UsePrivilegeSeparation": Disminuye la posibilidad de escalar de privilegios durante el inicio de sesión ejecutando este proceso de manera segura. Valor por defecto: `yes`.
				· "AllowUsers": Usuarios capaces de iniciar sesión, se pueden usar los comodines `*` y `?` de los patrones GLOB, para especificar múltiples se deben separar por espacios. Por defecto todos pueden iniciar sesión.
				· "PermitRootLogin": Permite conectarse con "root". Valor por defecto: `yes`.
				· "PasswordAuthentication": Habilita el inicio de sesión por contraseña. Valor por defecto: `yes`.
				· "PermitEmptyPasswords": Habilita el uso de contraseñas vacías. Valor por defecto: `no`.
				· "PubkeyAuthentication": Permite conectarse usando llaves SSH. Valor por defecto: `yes`.
				· "IgnoreRhosts": Deshabilitar `~/.rhosts` y `~/.shosts`, no afecta a `/etc/hosts.equiv` y a `/etc/ssh/shosts.equiv`. Valor por defecto: `yes`.
				· "ChrootDirectory": Establece como raíz la carpeta especificada, todos lo componentes de la ruta deben ser propiedad de `root` y ser solo editables por él, se pueden usar los comodines `%h` para hacer referencia a la carpeta principal del usuario y `%u` para hacer referencia al nombre del usuario. La carpeta espeficada debe poseer algunos elementos básicos del sistema para el funcionamiento de la sesión o usar la opción `ForceCommand internal-sftp` para que la sesión solo soporte el trabajo de archivos. Despues de cambiar la raíz, "sshd" intentará ir a la carpeta principal del usuario, por lo que se debe modificar en el archivo `/etc/passwd` y es recomendable asignarle el shell `/usr/sbin/nologin`.

			Autenticación:
				· "LoginGraceTime": Tiempo disponible para iniciar sesión. Valor por defecto: `120`.
				· "MaxAuthTries": Intentos para iniciar sesión. Valor por defecto: `6`.
				· "MaxSessions": Cantidad de sesiones permitidas por host. Valor por defecto: `10`.
				· "MaxStartups": Cantidad de conexiones simultaneas. Valor por defecto: `10`.
				· "AuthorizedKeysFile": Ruta a los archivos que contienen las llaves públicas, se pueden usar los comodines `%h` para hacer referencia a la carpeta principal del usuario y `%u` para hacer referencia al nombre del usuario, para especificar múltiples archivos se deben separar por espacios. Valor por defecto: `%h/.ssh/authorized_keys`.

			Otros:
				· "TCPKeepAlive": Envia señales al servidor para mantener la sesión activa, en caso de que se interrumpa el envio, se perderá la conexión y se detendrá el proceso en el servidor, si se deshabilita esta opción, nunca se detendrá el proceso en el servidor. Valor por defecto: `yes`.
				· "Banner": Muestra el contenido del archivo especificado al iniciar sesión. Desactivado por defecto.
				· "DebianBanner": Muestra información de la distribución. Valor por defecto: `yes`.
				· "PrintMotd": Muestra el contenido del archivo `/etc/motd` al iniciar sesión. Valor por defecto: `yes`.
				· "PrintLastLog": Muestra el último inicio al entrar. Valor por defecto: `yes`.
				· "AcceptEnv": Variables de entorno a transferir desde el cliente, se pueden usar los comodines `*` y `?` de los patrones GLOB, para especificar múltiples se deben separar por espacios. Sin definir por defecto.
				· "Compression": Habilita la compresión de contenido. Valor por defecto: `yes`.

		Opciones obsoletas:
			· "KeyRegenerationInterval"
			· "RhostsRSAAuthentication"
			· "RSAAuthentication"
			· "ServerKeyBits"
