Windows:
	Ejecutar:
		· Administrador de programas al inicio:
			msconfig

		· Asistencia remota:
			gpedit.msc
			
		· Consola MSDos:
			cmd

	Recuperación:
		· Reparar boot:
			XP:
				fixmbr
				fixboot

			Windows Vista, 7 y 8.1:
				bootrec.exe [/fixmbr] | [/fixboot] | [/scanos] | [/rebuildbcd]

				Parámetros:
					/fixmbr: Reescribe el MBR por uno compatible con Windows.
					/fixboot: Reescribe el sector de boot de Windows.
					/scanos: Busca instalaciones de Windows en el disco duro.
					/rebuildbcd: Busca y agrega instalaciones de Windows al BCD.

		· Verificar sistema archivos:
			chkdsk <disco>: /r

			Parámetros:
				<disco>: La letra de la partición a verificar.
	
	Consola:
		. Conexiones:
			. IP:
				ipconfig (/all para obtener informacion mas detallada)
				
			. Ping:
				ping "Direccion IP"
				
		. Verificar validacion:
			. Windows 8.1:
				slmgr -xpr

Linux:
	Leyenda:
		comando {opción | otraOpción} [-p | --parámetro] <dato>...

			Escritura común: Debe escribirse exactamente igual. 
			{}: Secciones.
			[]: Opcional. 
			|: Opciones a elegir.
			-: Parámetros de una letra.
			--: Parámetros descriptivos.
			<>: Dato ingresado por el usuario.
			...: Repetición.

	Prompt:
		Indica al usuario que el terminal está esperando instrucciones.
			usuario@Computador:/home/usuario/Descargas$

			usuario: Nombre del usuario activo.
			Computador: Nombre del computador.
			/home/usuario/Descargas: Carpeta activa.
			$ o #: Nivel de privilegios, $ privilegios limitados y # privilegios de superusuario.

	Variables de entorno:
		Acceder:
			$ $<variable>

		Crear / Asignar valor:
			$ export <variable> = <valor>

		Eliminar:
			$ unset <variable>

		Mostrar:
			$ env  # Todas
			$ echo $<variable>  # Especifica

			Lista de variable:
				EDITOR: Editor de texto predeterminado.
				HISTFILE: Archivo donde se guarda el historial de comandos del usuario activo.
				HISTFILESIZE: Cantidad de líneas que guardará el archivo del historial de comandos.
				HISTSIZE: Cantidad de líneas que contiene el archivo del historial de comandos.
				HOME: Ruta a la carpeta personal del usuario activo.
				LANG, LANGUAGE, LC_ALL: Idioma y localización.
				OLDPWD: Ruta anterior.
				PAGER: Programa usado para mostrar el contenido de los archivo.
				PATH: Carpetas donde se buscarán los ejecutables para evitar escribir su ubicación.
				PWD: Ruta actual.
				RANDOM: Valor aleatorio.
				SHELL: Shell que ejecutará los comandos.
				USER: Usuario actual.
				USERNAME: Usuario de la sesión.

	Comandos:
		· Aplicaciones y paquetes:
			APT (repositorios):
				Actualizar:
					Lista de paquetes:
						# apt update

						Nota: Para simular la actualización de la lista de paquetes se usa el comando:
							$ apt-get -s update

					Paquetes:
						# apt upgrade

						# apt-get [-s | -d | --no-download | --print-uris] [-y | --assume-no] upgrade

						Parámetros:
							-s: Emula la actualización, pero no hace ningún cambio en el sistema.
							-d: Descarga pero no actualiza los paquetes, normalmente se guardan en "/var/cache/apt/archives/".
							--no-download: Intenta realizar la actualización con paquetes locales (en "/var/cache/apt/archives/").
							--print-uris: Muestra las direcciones de los paquetes en los repositorios en lugar de descargarlos.
							-y: Acepta cualquier confirmación.
							--assume-no: Niega cualquier confirmación.

						Nota: Para actualizar solo un paquete se debe usar el comando:
							apt-get install <paquete> --only-upgrade

				Buscar:
					$ apt-cache [-n] search <expresiónRegular>

					Parámetros:
						-n: Busca coincidencias solo en el nombre del paquete.

					Nombre:
						$ apt list [--installed | --upgradable | -a] "<patrónGlob>"

						Parametros:
							--installed: Busca en los paquetes instalados.
							--upgradable: Busca en los paquetes actualizables.
							-a: Muestra todas las versiones.

						$ apt-cache pkgnames [<nombrePaquete>]

					Descripción:
						$ apt search <cadena>

				Descargar:
					$ apt-get download <paquete>

					Nota: Equivalente a "apt-get -d install <paquete>", solo que el paquete no se descarga en "/var/cache/apt/archives/" sino en la carpeta activa.

				Desinstalar:
					# apt-get [-s] [-y | --assume-no] remove | purge <paquete>

					Parámetros:
						-s: Emula la desinstalación, pero no hace ningún cambio en el sistema.
						-y: Acepta cualquier confirmación.
						--assume-no: Niega cualquier confirmación.

					Nota: "purge" a demás de desinstalar, elimina los archivos de configuración del paquete.

				Detallar:
					$ apt show <paquete>
					$ apt-cache show <paquete>  # Más detalle
					$ apt-cache showpkg <paquete>  # Aun más detalle

					Dependencias:
						$ apt-cache depends <paquete>

					Dependencias inversas:
						$ apt-cache rdepends <paquete>

				Editar repositorios:
					# apt edit-sources

				Instalar:
					# apt-get [--install-suggests | --no-install-recommends] [-s | -d | --no-download | --print-uris] [--no-upgrade] [--reinstall] [-y | --assume-no] install <paquete>[=<versión>]

					Parámetros:
						--install-suggests: Instala los paquetes sugeridos.
						--no-install-recommends: No instala los paquetes sugeridos.
						-s: Emula la instalación, pero no hace ningún cambio en el sistema.
						-d: Descarga pero no instala los paquetes, normalmente se guardan en "/var/cache/apt/archives/".
						--no-download: Intenta realizar la instalación con paquetes locales (en "/var/cache/apt/archives/").
						--print-uris: Muestra las direcciones de los paquetes en los repositorios en lugar de descargarlos.
						--no-upgrade: Evita que se actualicen paquetes durante una instalación.
						--reinstall: Reinstala los paquetes especificados si se encuentran instalados.
						-y: Acepta cualquier confirmación.
						--assume-no: Niega cualquier confirmación.

					Nota: La lista de repositorios usados por APT normalmente se encuentra en "/etc/apt/source.list".

				Limpiar:
					Paquetes descargados:
						# apt-get clean | autoclean

						Nota: "clean" borra todos los paquetes descargados, "autoclean" solo los innecesarios.

					Paquetes innecesarios:
						# apt-get autoremove

				Reparar dependencias rotas:
					# apt-get -f install | remove

				Verificar dependencias rotas:
					# apt-get check

			DPKG (.deb):
				Arquitectura:
					Agregar:
						# dpkg --add-architecture <arquitectura>

						Valores:
							· amd64
							· i386

					Eliminar:
						# dpkg --remove-architecture <arquitectura>

					Mostrar:
						$ dpkg --print-architecture | --print-foreign-architectures

						Nota: "--print-architecture" muestra solo la arquitectura predeterminada, "--print-foreign-architectures" muestra las arquitecturas alternativas agregadas.

				Buscar:
					$ dpkg-query -l <patrónNombre>

					Archivos creados por la instalación:
						$ dpkg-query -S <patrónNombre>

				Configurar:
					# dpkg --configure <patrónNombre> | -a

					Nota: Si se especifica "-a", dpkg buscará y configurará cualquier paquete que aún no haya sido configurado.

				Desinstalar:
					# dpkg -r | -P [--force-<opción> | --refuse-<opción>] <patrónNombre>

					Parámetros:
						-r: Desinstala el paquete.
						-P: Desinstala y elimina los archivos de configuración del paquete.
						--force-<opción> | --refuse-<opción>:
							· not-root: Desinstala sin privilegios de superusuario.
							· remove-essential: Elimina aunque sea importante para el sistema.

							Nota: "--refuse" niega la opción.

				Detallar:
					$ dpkg-query -s <paquete>

					Formato personalizado:
						$ dpkg-query -Wf '<formato>' <patrónNombre>

						Formato:
							Escapes:
								\<caracter>: Elimina el significado especial de <caracter>.
								\n: Salto de línea.
								\r: Retorno de línea.
								\t: Tabulación.

							Datos:
								${<campo>}

								· Archivos de configuración: Conffiles.
								· Arquitectura: Architecture.
								· Categoría: Section.
								· Conflictos: Conflicts.
								· Dependencias: Depends.
								· Descripción: Description.
								· Errores: Bugs.
								· Estado: Status.
								· Mantenedor: Maintainer.
								· Nombre: Package.
								. Paquetes que remplaza: Replaces.
								· Paquetes recomendados: Suggests.
								· Prioridad: Priority.
								· Sitio oficial: Homepage.
								· Uso de disco: Installed-Size.
								· Versión: Version.

						Nota: Si no se usa "-f" el formato predeterminado es: Nombre Versión.

				Instalar:
					# dpkg -i [-R] [-G] [-E] [--no-debsig] [--force-<opción> | --refuse-<opción>] <patrónGlob>

					Parámetros:
						-R: Método recursivo para instalar paquetes desde una carpeta.
						-G: Evita la instalación de versiones anteriores.
						-E: Omite el paquete si existe la misma versión instalada.
						--no-debsig: Evita la verificación de la firma del paquete.
						--force-<opción> | --refuse-<opción>:
							· architecture: Instala sin importar la arquitectura.
							· break: Instala aunque se rompa otro paquete.
							· conflicts: Instala aunque existan conflictos.
							· depends: Evita que las dependencias detengan la instalación, solo muestra alertas.
							· depends-version: Omite la verificación de versiones en las dependencias.
							· downgrade: Instala aunque exista un paquete mas actualizado.
							· not-root: Instala sin privilegios de superusuario.
							· overwrite: Reinstala un paquete.

							Nota: "--refuse" niega la opción.

				Reparar:
					# dpkg-reconfigure [--force] <paquete> | -a

					Parámetros:
						--force: Intenta reconfigurar un paquete aunque este roto.
						-a: Reconfigura todos los paquetes.

				Nota: Es posible simular todas las tareas de dpkg con el parámetro "--no-act".

			Alien (conversor de paquetes):
				# alien [-d | -r | -t | --to-slp | -p] [-i] [-v] <patrónGlob>

				Parámetros:
					-d: Convierte a .deb.
					-r: Conviente a .rpm.
					-t: Convierte a .tgz.
					--to-slp: Convierte a .slp.
					-p: Conviente a .pkg

					-i: Instala automaticamente despues de convertir el paquete, al finalizar la instalación se elimina el paquete generado.
					-v: Muestra el proceso de conversión.

				Nota: Se debe instalar el paquete "alien".

		. Dispositivos:
			Nombre del equipo:
				$ hostname [-I]

				# hostname [-I] | [<nombreEquipo>]

				Parámetros:
					-I: Muestra la dirección IP.

			Información del equipo:
				Kernel y sistema operativo:
					$ uname {[-s] [-n] [-r] [-v] [-m] [-p] [-i] [-o]} | [-a]

					Parámetros:
						-s: Nombre del kernel.
						-n: Nombre de red del equipo.
						-r: Número de liberación del kernel.
						-v: Versión del kernel.
						-m: Arquitectura del kernel
						-p: Tipo de procesador.
						-i: Información del hardware.
						-o: Sistema operativo.
						-a: Todas las opciones.

				RAM y Swap:
					free [-h] [-o] [-s <segundos> [-c <cantidadVerificaciones>]] [-t]

					Parámetros:
						-h: Da formato a las medidas obtenidas.
						-o: Omite el buffer.
						-s <segundos>: Realiza la verificación cada <segundos> segundos.
						-c <cantidadVerificaciones>: Limita la cantidad de verificaciones realizadas por -s.
						-t: Muestra al final la sumatoria de las memorias.

			Tiempo de actividad del Sistema Operativo:
				$ uptime

		· Sistema de archivos:
			Borrar:
				$ rm [-r] [-i | -f] [-v] <patrónGlob>

				Parámetros:
					-r: Para eliminar carpetas y su contenido.
					-i: Realiza una confirmación por cada archivo y carpeta que intente eliminar.
					-f: Forza la eliminación.
					-v: Muestra cada archivo y carpeta que se elimine.

			Buscar:
				$ find <ruta> [<otraRuta>] [-not] <criterio> [[-or] [-not] <otroCriterio>] <acción>

				Parámetros:
					Operadores:
						(): Prioridad.
						-not: Negación.
						-or: OR lógico.

					Criterios de búsqueda:
						· Grupo: -group <grupo>
						· Nombre:
							Sensible a mayúsculas y minúsculas: -name <patrónGlob>
							No sensible a mayúsculas y minúsculas: -iname <patrónGlob>
						· Número de enlaces: -links [- | +]<númeroEnlaces>
						· Permisos:
							-perm [-]<permisos>

							Nota: Si no se utiliza el "-", solo se encontrarán archivos con exactamente los permisos indicados. Ejemplo:
								find . -perm u=r -> Archivos con u=r,g=,o=
								find . -perm -u=r -> Archivos con al menos u=r

							Alternativas:
								Editable: -writable
								Ejecutables: -executable
								Legibles: -readable
						· Ruta: -path <patrónGlob>
						· Tamaño: -size [- | +]<peso><medida>
							c: Bytes
							k: Kilobytes
							M: Megabytes
							G: Gigabytes
						· Tiempo:
							Acceso:
								Minutos: -amin [- | +]<minutos>
								Dias: -atime [- | +]<dias>
							Modificacion:
								Minutos: -mmin [- | +]<minutos>
								Dias: -mtime [- | +]<dias>
						· Tipo: -type <tipo>
							d: Carpeta
							f: Archivo
							l: Enlace simbólico
						· Usuario: -user <usuario>
						· Vacio: -empty

					Acciones:
						Ejecutar: -exec <comando> \;
						Eliminar: -delete
						Mostrar: -print

				Notas:
					· Si se establece como ruta ".", se realizará la busqueda en la carpeta actual.
					· Es posible establecer multiples criterios de busqueda.
					· Si no se especifica un operador se asume que se debe tratar como -and.
					. El número en los criterios puede expresarse con + o -, ejemplo:
						find . -type f -amin 60 -> Archivos a los que accedieron hace exactamente una hora
						find . -type f -amin -60 -> Archivos a los que accedieron hace menos de una hora
						find . -type f -amin +60 -> Archivos a los que accedieron hace más de una hora
					· Si no se especifica una acción, se asume que se debe usar -print.

			Comprimir/Descomprimir:
				tar {{{c | r | u } [-j | -z] [-p] [--remove-files]} | t | {x [-a] [-C <carpeta>] [--overwrite]}} [-v] [--exclude=<patrónGlob> | -X <archivo>] [-w] -f <archivoTar>.tar[.bz2 | .gz] <patrónGlob> | -T <archivo>

				Modos:
					c: Crea un nuevo paquete.
					r: Agrega los archivos a un paquete existente.
					u: Agrega solo los paquetes nuevos/actualizados a un paquete existente.

					t: Muestra el contenido del paquete.

					x: Extrae el contenido del paquete, se omite <patrónGlob>.

				Parámetros:
					-j: Comprime con el algorito bzip2, se debe agregar ".bz2" al paquete.
					-z: Comprime con el algorito gzip, se debe agregar ".gz" al paquete.
					-p: Mantiene los permisos.
					--remove-files: Elimina los archivos originalos luego de ser agregados al paquete.
					-a: Identifica automaticamente el algoritmo con que el que se comprimió.
					-C <carpeta>: Extrae el contenido del paquete en la carpeta especificada.
					--overwrite: Sobrescribe los archivos al extraerlos.
					-v: Muestra los archivos procesados.
					--exclude=<patrón>: No procesa las coincidencias con el patrón.
					-X <archivo>: No procesa las coincidencias con los patrones en <archivo>.
					-w: Solicita confirmación para cada acción.
					-T <archivo>: Extrae o empaqueta solo las coincidencias de los patrones dentro de <archivo>.

			Concatenar:
				$ cat [-b | -n] [-A] <patrónGlob> | <archivo> [<otroArchivo> [<otroArchivo]]

				Parámetros:
					-b: Enumera solo las líneas con contenido.
					-n: Enumera todas las líneas
					-A: Muestra todos los caracteres especiales.
						$: Final de linea.
						^I: Tabulación.

				Nota: Puede ser utilizado para mostrar el contenido de un archivo.

			Copiar:
				$ cp [-r] [-f | -i | -n] [-s] [-u] [-v] <patrónGlob> <destino>

				Parámetros:
					Metodos:
						-f: Sobrescribe cualquier coincidencia que encuentre.
						-i: Realiza una confirmación para cualquier coincidencia que encuentre.
						-n: Omite cualquier archivo que coincida.

					Opciones:
						-r: Para copiar carpetas y su contenido.
						-s: Crea un enlace simbólico en lugar de copiar.
						-u: Copia solo si la coincidencia encontrada es mas antigua o si no existe el archivo.
						-v: Muestra el nombre de los archivos/carpetas copiados.

			Crear:
				· Archivo:
					$ touch [-a | -m] [-c] [-r <otroArchivo> | -d <fecha>] <archivo>

					Parametros:
						-a: Cambia solo la fecha de acceso.
						-m: Cambia solo la fecha de modificación.
						-c: Si <archivo> no existe, evita crearlo.
						-d <fecha>: Usa <fecha> para asignar los valores, ej: touch -d "1992-04-30 22:00" M.txt
						-r <otroArchivo>: Usar <otroArchivo> como referencia para asignar los valores.

				· Carpeta:
					$ mkdir [-m <modo>] [-p] [-v] <carpeta>

					Parametros:
						-m <modo>: Asigna la permisología especificada a las carpetas creadas.
						-p: Si los ancestros especificados no existen serán creados.
						-v: Muestra en pantalla las carpetas que se crean.

				. Enlaces:
					$ ln [-s] | [-P] <origen> <enlace>

					Parámetros:
						-s: Enlace simbólico.
						-P: Enlace rigido.

					Nota: Para una carpeta se debe especificar <ruta/origen/>.

			Editar:
				$ <comandoEditor> <archivo>

				Nota: Normalmente se usa "nano" como editor de texto desde terminal.

			Información:
				Contenido:
					· Archivo:
						$ more | less <archivo>

					· Carpeta:
						$ ls [-l [-h] | -C [-1] | -m] [-a] [-A] [-F] [-R] [-L] [-s] [<patrónGlob>]

						Parámetros:
							Formatos de salida:
								-l: Muestra el contenido de la carpeta en una lista con detalles.
									-h: Da formato al tamaño de los archivos mostrados por -l o -s.
								-C: Muestra el contenido de la carpeta en columnas (solo nombres).
									-1: La salida de -C se muestra en una sola columna.
								-m: Muestra el contenido de la carpeta separados por ",".

							Opciones:
								-a: Muestra todos los archivos, incluso los ocultos.
								-A: Tiene el mismo comportamiento que -a pero excluye "." y ".." de la salida.
								-F: Agrega un posfijo según el tipo de archivo.
									/: Carpetas.
									@: Enlaces.
								-R: Muestra el contenido de las carpetas decendientes.
								-L: Sustituye los enlaces por el archivo/carpeta que referencia.
								-s: Muestra el tamaño de los archivos/carpetas antes de su nombre.
								-d: Hace que las carpetas de comporten como archivos y asi evitar mostrar sus contenidos.

				Uso de disco:
					$ du [-a | -s] [-c] [-h] [-X <archivo>] [--exclude=<patrónGlob>] [--time] <carpeta> | <archivo>

					Parámetros:
						Formatos de salida:
							-a: Muestra el peso de cada archivo contenido en las carpetas.
							-s: Muestra la sumatoria del uso del disco.
						
						Opciones:
							-c: Muestra el total al final.
							-h: Da formato al tamaño.
							-X <archivoX>: Omite archivos según la lista de patrones GLOB dentro de "archivoX".
							--exclude=<patrónGlob>: Omite archivos según el patrón indicado.
							--time: Última fecha de modificación.

			Mover / Renombrar:
				$ mv [-r] [-f | -i | -n] [-u] [-v] <patrónGlob> <destino> | <nuevoNombre>

				Parámetros:
					Metodos:
						-f: Sobrescribe cualquier coincidencia que encuentre.
						-i: Realiza una confirmación para cualquier coincidencia que encuentre.
						-n: Omite cualquier archivo que coincida.

					Opciones:
						-r: Para mover carpetas y su contenido.
						-u: Mueve/renombra solo si la coincidencia encontrada es mas antigua o si no existe el archivo.
						-v: Muestra el nombre de los archivos/carpetas movidos/renombrados.

				Nota: Si se encuentran coincidencias entre carpetas, se mezcla su contenido.

			Navegar:
				· Carpeta actual:
					$ pwd

				· Cambiar carpeta:
					$ cd [<carpeta> | .. | - | ~<usuario>]

					Parámetros:
						..: Se trasladará a la carpeta ancestro de la carpeta actual.
						-, $OLDPWD: Se trasladará a la carpeta en la que se encontraba antes de ejecutar el comando "cd".
						~, $HOME: Se trasladará al directorio personal del usuario activo.
						~<usuario>: Se trasladará al directorio personal del usuario indicado.

					Nota: Si no se especifica algún parámetro, tendrá el mismo comportamiento que ~ y $HOME.

			Ocultar:
				Se debe agregar "." al inicio del archivo o carpeta.

				· Archivo:
					Nuevo:
						$ touch .<archivo>

					Existente:
						$ mv <archivo> .<archivo>

				· Carpeta:
					Nueva:
						$ mkdir .<carpeta>
					
					Existente:
						$ mv <carpeta> .<carpeta>

			Permisos:
				$ chmod [-c | -v] [-f] [-R] --reference=<archivo> | <modo> <patrónGlob>

				Parámetros:
					-c: Muestra los archivos/carpetas a los que se les modificó el modo de bits.
					-v: Muestra todos los archivos/carpetas procesados.
					-f: Omite mensajes de error.
					-R: Aplica el modo de bits a todo el contenido de una carpeta.
					--reference=<archivo>: Aplica el modo de bits que posea el archivo especificado.

				Modos:
					Simbólico:
						{u | g | o | a}{= | + | -}<permisos>

						· Usuarios:
							u: Propietario.
							g: Grupo.
							o: Otros.
							a: Todos.

						· Operadores:
							=: Establece el modo de bits como se especifica.
							+: Agrega el bit especificado sin modificar los demás.
							-: Elimina el bit especificado sin modificar los demás.

						· Permisos:
							r: Lectura.
							w: Escritura.
							x: Ejecución/Lectura de carpeta.
							X: Ejecución solo para las carpetas y archivos que ya tengan permisos de ejecución.
							t: Restricción de eliminación/modificación a usuarios que no sean el propietario.

						Nota: Pueden especificarse diferente modos de bits a diferentes usuarios separados por comas.

						Ejemplos:
							chmod u=rw <patrónGlob> -> Solo el propietario podra leer y escribir el archivo/carpeta.
							chmod o+r <patrónGlob> -> Ahora los usuarios que no pertenezcan a algún grupo del computador podran leer el archivo/carpeta.
							chmod -t <patrónGlob> -> Elimina la restricción de eliminación/modificación.

					Octal:
						[<octalPermisosExtras>]<octalPropietario><octalGrupo><octalOtros>

						Cada octal representa 3 bits:	
							1. Lectura.
							2. Escritura.
							3. Ejecución.

						Valores:
							0 -> 000
							1 -> 001
							2 -> 010
							3 -> 011
							4 -> 100
							5 -> 101
							6 -> 110
							7 -> 111

						Ejemplos:
							chmod 600 <patrónGlob> -> Solo el propietario podra leer y escribir el archivo/carpeta.
							chmod 777 <patrónGlob> -> Habilita todos los permisos.
							chmod 1744 <patrónGlob> -> Agrega la restricción de eliminación/modificación, da al propietario todos los permisos y el resto de usuarios solo podrán leer el archivo/carpeta.

				· Cambiar propietario:
					$ chown [-c | -v] {[--from=<propietario> | <grupo> | <propietario>:<grupo>]} [-f] [-R] {--reference=<archivo> | <propietario> | <propietario>:<grupo>} <patrónGlob>

					Parámetros:
						-c: Muestra los archivos/carpetas a los que se les modificó el propietario.
						-v: Muestra todos los archivos/carpetas procesados.
						--from: Solo afecta archivos/carpetas que tengan actualmente el propietario/grupo especificado.
						-f: Omite mensajes de error.
						-R: Aplica el cambio de propietario a todo el contenido de una carpeta.
						--reference=<archivo>: Asiga el propietario y grupo que posea el archivo especificado.

		· Utilidades:
			Salida de texto:
				$ echo [-n] [-e] "<cadena> $(<comando>)"

				Parámetros:
					-n: Evita el salto de linea predeterminado del comando.
					-e: Habilita el escape de caracteres en <cadena>.
						\<caracter>: Evita que <caracter> tenga un comportamiento especial.
						\a: Alerta sonora.
						\b: Borrar un caracter.
						\n: Salto de linea.
						\r: Retorno de carro.
						\t: Tabulador.
						\v: Tabulador vertical.

				Nota: Para que "echo" escriba salidas de comandos se debe usar $(<comando>).

	Otros:
		· Ejecutar varios comandos en una linea:
			<comando>; <otroComando>  # Ejecuta <comando> y luego <otroComando>.
			<comando> && <otroComando>  # Ejecuta <comando>, si se ejecutó correctamente entonces <otroComando> es ejecutado.
			<comando> || <otroComando>  # Ejecuta <comando>, solo si ocurre un error entonces se ejecuta <otroComando>.

		. Redireccionar salidas
			Comando -> archivo:
				$ <comando> > <archivo>  # Sobrescribe el contenido del archivo con la salida del comando, si no existe <archivo> se crea automaticamente.
				$ <comando> >> <archivo>  # Agrega la salida del comando al final del contenido archivo.

			Comando -> comando:
				$ <comando> | <otroComando>  # Ejecuta <comando> y redirecciona su salida a <otroComando>
				$ <comando> $(<otroComando>)  # Ejecuta <otroComando> y usa su salida para ejecutar <comando>

				badblocks -wvs /dev/sdxx

			Nota: vs -> Para mostrar detalles del progreso
